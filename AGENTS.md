# Agent Project Guidance

Root instruction map for coding agents working on `expo-splash-full-screen`. Keep this file short. The nearest `AGENTS.md` applies to the files being edited.

## Highest-Priority Rules

- Use `bun` and `bunx`. Do not use `npm`, `npx`, or `yarn`.
- Use `bun run typecheck` and `bun run lint` before finishing. Do not call `tsgo`, `oxlint`, or `oxfmt` directly.
- Do not disable lint rules to make a change pass. If a rule is genuinely wrong, stop and ask.
- Do not add features, knobs, or abstractions beyond what the task asks. No backwards-compat shims. No config options with no user.
- When you change one side of a plugin-native pair (Info.plist key, Android `res/values` name, drawable/imageset name, JS API shape), change the paired side in the same commit.
- Do not rebuild the consumer app (`expo run:android`, `expo run:ios`). The user runs those. Verify via prebuild + inspect generated artifacts.

## Repo Map

- `src/` — TypeScript binding. `requireNativeModule('SplashScreenModule')`. Public surface in `index.ts` (default export `SplashScreen` + `HideOptions`).
- `plugin/src/` — Expo config plugin. Writes imagesets, `splashscreen.xml`, Info.plist keys, storyboard, and rewrites `windowBackground` on `AppTheme` + `Theme.App.SplashScreen`.
- `android/src/main/java/expo/modules/splashfullscreen/` — Kotlin native. `SplashScreenOverlay` singleton mounts a transparent `Dialog` from `ReactActivityLifecycleListener.onCreate`.
- `ios/` — Swift native. `SplashScreenOverlay.shared` adds a `UIView` to the key window from `ExpoAppDelegateSubscriber.didFinishLaunchingWithOptions`.
- `expo-module.config.json` — autolinking manifest (iOS class names + fully-qualified Android FQNs).
- `ios/ExpoSplashFullScreen.podspec` — CocoaPods spec; name matches the Swift module name used in autolink output.
- `build/`, `plugin/build/` — tsgo output. Regenerated by `bun run build`.

## Architecture Invariants

- Two overlay layers: optional icon over a full-bleed image. Both are siblings of a root with the configured `backgroundColor`. Icon cross-fades out as full-bleed fades in.
- Single linear timeline: `fadeIn → hold(iconDisplayMs) → crossfade → hold(fullscreenHoldMs) → fadeOut`. `hide()` **defers** the fade-out until the minimum elapses — never truncate the brand moment.
- Easing per phase (keep in sync on both platforms):
  - `fadeIn`: ease-out. `DecelerateInterpolator` / `.curveEaseOut`.
  - `crossfade`: ease-in-out. `PathInterpolator(0.77, 0, 0.175, 1)` / `.curveEaseInOut`.
  - `fadeOut`: ease-in. `AccelerateInterpolator` / `.curveEaseIn`.
- Android: bypass `Theme.SplashScreen` icon-only enforcement with a transparent `Dialog`. Do not try to inflate content into the system SplashScreen — Android 12+ clips to a circle.
- iOS: `SplashScreen.storyboard` carries only the pre-overlay background color + optional centered icon, so the OS → overlay handoff is seamless. The storyboard template is patched at prebuild from plugin props — do not hardcode colors.
- Plugin scripts run at prebuild. Resolve user asset paths relative to `modRequest.projectRoot`. Stay inside `platformProjectRoot` when writing.
- Info.plist keys use the `Splash*` prefix (e.g. `SplashFadeIn`, `SplashBackgroundColor`). Android resource names use `splash_*`. Renaming one side without the other silently reverts runtime behavior to defaults.

## Tooling

- `bun install` — installs deps. `prepare` runs `clean` + `build` automatically.
- `bun run build` — tsgo compiles `src/` → `build/` and `plugin/src/` → `plugin/build/`.
- `bun run typecheck` — tsgo `--noEmit` on both projects.
- `bun run lint` — oxlint.
- `bun run format` / `bun run format:check` — oxfmt. Respects `.gitignore` (`build/`, `plugin/build/`, `node_modules/` excluded).
- `bun pm pack` — emits `expo-splash-full-screen-<version>.tgz` for consumer installs.

## Consumer Workflow

Consumer apps (`/Users/thanglb/workspace/aaf-splash-test` is the reference) install via tarball:

```json
"expo-splash-full-screen": "file:../expo-splash-full-screen/expo-splash-full-screen-0.1.0.tgz"
```

Do **not** switch to `file:../expo-splash-full-screen` (directory). Bun `file:` directory deps create symlinks; Expo autolinker's `scanFilesRecursively` skips symlinked `.kt` files → `packages: []` → `ReactActivityLifecycleListener` never registers → no splash.

After editing lib code:

1. In lib: `bun run build && bun pm pack`.
2. In consumer: move `node_modules` + `bun.lock` aside (`mv ... /tmp/...-$(date +%s)`), then `bun install`. Bun's tarball cache can otherwise serve stale content when the version did not change.
3. `bunx expo prebuild --clean` (required after any plugin change — regenerates `splashscreen.xml`, Info.plist, imagesets, drawables, storyboard, app theme).

## Verification

No automated test suite. Verify changes by:

1. `bun run typecheck && bun run lint && bun run format:check` in the lib.
2. Repack + reinstall + `bunx expo prebuild --clean` in the consumer.
3. Inspect:
   - `android/app/src/main/res/values/splashscreen.xml` — all `splash_*` keys present with expected values.
   - `android/app/src/main/res/values/styles.xml` — `AppTheme` and `Theme.App.SplashScreen` both have `android:windowBackground = @color/splash_background`.
   - `android/app/src/main/res/drawable-*/splash_*.png` — scaled fullscreen + icon.
   - `ios/<App>/Info.plist` — `Splash*` keys.
   - `ios/<App>/Images.xcassets/Splash{FullScreen,Icon}.imageset/` — 1x/2x/3x.
   - `ios/<App>/SplashScreen.storyboard` — backgroundColor matches plugin prop; icon present iff `iconSplash.ios`.
4. `bunx expo-modules-autolinking resolve --platform android --json` — `packages[]` must contain `expo.modules.splashfullscreen.SplashScreenPackage`. Empty = autolink broken.
5. User runs `bunx expo run:android` / `run:ios` on their side.

## Lint / Format Policy

- Fix root cause first.
- `// oxlint-disable-next-line <rule> -- <reason>` is last resort. `-- <reason>` mandatory.
- Do not loosen `.oxlintrc.json` for one file. If a rule is systemically wrong, stop and ask.
